Privacy Policy

Last updated: 2026-05-27

Data Collection

QRecorder collects the following data solely for the purpose of recording and replaying browser interactions for test automation:

  • HTTP Request Data: URLs, headers, request bodies, response bodies, status codes, and timing information for XHR/Fetch requests made by pages you choose to record.
  • UI Interaction Events: Click, type, select, and other DOM events on pages you actively record, including element selectors and input values.
  • Authentication Token: A JWT token you provide when connecting to a QRecorder backend server, used only for API authentication.

How Data Is Used

  • All collected data is used exclusively for creating test scenarios and enabling test replay.
  • Data is stored locally in your browser's chrome.storage.local and is never sent to any third party.
  • If you choose to connect to a QRecorder backend server, recorded data is sent only to that server at your direction.

Data Storage

  • Recorded data remains in your local browser storage until you delete it or clear extension data.
  • When you save a scenario to a backend server, that server becomes responsible for the data it receives.

Permissions

This extension requires the following permissions and why:

  • <all_urls> / webRequest: To capture HTTP requests from pages you choose to record. Recording is only active when you explicitly start it.
  • tabs: To navigate tabs during replay and to identify the correct tab for recording.
  • storage: To save recordings and settings locally.
  • activeTab: To interact with the currently active tab when starting/stopping recording.

Third-Party Sharing

QRecorder does not share any data with third parties. It does not use analytics SDKs, tracking pixels, or external data processors.

Contact

For questions about this privacy policy, please contact the extension developer through the support channel listed in the extension's store listing.
